# Whitepaper Quality Checklist

Use this checklist to evaluate whitepaper quality during review. Each item should be checked before the paper is considered final.

## Structure and Organization

* [ ] Paper follows the standard section order for its type

* [ ] Table of Contents present (if paper exceeds 6 pages) with working links

* [ ] Abstract OR Executive Summary present (appropriate to audience)

* [ ] Introduction establishes urgency with data before presenting any solution

* [ ] Roadmap paragraph tells reader what to expect

* [ ] Each section logically "earns" the next — clear narrative flow

* [ ] Conclusion summarizes without introducing new information

* [ ] All appendices and glossary items are referenced in the body text

## Evidence and Credibility

* [ ] Every factual claim is backed by a citation, benchmark, or case study

* [ ] Statistics come from primary sources (not secondary blog summaries)

* [ ] No statistics older than 3 years unless foundational research

* [ ] Case studies include specific, quantified metrics

* [ ] Results are quantified concretely ("reduced by 67%", not "significantly improved")

* [ ] Methodology section present (for research-based papers)

* [ ] Limitations and constraints are acknowledged

* [ ] For technical papers: results are reproducible with enough implementation detail

## Writing Quality

* [ ] Sentences average under 14 words

* [ ] No buzzwords: "leveraging," "best-in-class," "industry-leading," "synergies," "state-of-the-art"

* [ ] No undefined company-specific jargon

* [ ] Industry-standard terminology used correctly

* [ ] Third-person voice maintained (for formal/technical papers)

* [ ] No passive-voice obfuscation hiding who did what

* [ ] Reader-first framing: paper is about what the READER gets, not what the company announces

## Reader-First Test

Apply this test to the introduction and each section:

**Poor framing:** "This white paper introduces ABC Company's new freight service."
**Good framing:** "This white paper discusses how to choose a freight service company that best fits your needs."

* [ ] Title emphasizes reader value, not company announcements

* [ ] Introduction focuses on the reader's problem, not the company's product

* [ ] Product/solution mentions appear only after credibility is established

* [ ] Call to action feels earned, not forced

## Visual Elements

* [ ] All figure captions placed BELOW figures

* [ ] All table captions placed ABOVE tables

* [ ] All visuals referenced in text by number ("Figure 3"), not position ("below")

* [ ] All visuals have: numbered caption, descriptive title, labeled axes/legends

* [ ] External data sources attributed

* [ ] Diagrams used for architecture/flows, tables for comparisons, charts for trends

* [ ] Color-blind-safe palette used

* [ ] Alt text provided for accessibility

## Citations and References

* [ ] Citation style consistent throughout (IEEE, APA, MLA, or Chicago)

* [ ] All in-text citations have corresponding reference list entries

* [ ] All reference list entries are cited in-text

* [ ] URLs verified as accessible

* [ ] Whitepaper citations formatted correctly for the chosen style

## Formatting and Design

* [ ] Company logo and branding on cover page

* [ ] Page numbers on every page

* [ ] Publication date and version number on cover

* [ ] Consistent heading hierarchy (H1, H2, H3)

* [ ] Hierarchical numbering used (1.0, 1.1, 2.0)

* [ ] Generous white space and margins

* [ ] No dense text blocks longer than 5 sentences without a break

* [ ] Consistent typography throughout

## Type-Specific Checks

### Pure Technical

* [ ] Architecture diagrams included

* [ ] Code snippets present with syntax highlighting

* [ ] Performance benchmarks with test methodology

* [ ] Implementation steps detailed enough for reproduction

### Problem/Solution

* [ ] Problem established before any solution is mentioned

* [ ] Pain points amplified with data

* [ ] Solution presented as logical conclusion of evidence

### Backgrounder/Product

* [ ] Competitive comparisons handled fairly

* [ ] Both features AND business outcomes covered

* [ ] Quantified ROI or TCO included

### Thought Leadership

* [ ] Original perspective present (not rehash of known information)

* [ ] Named author with established credentials

* [ ] No product promotion in the body

### Market Research

* [ ] Methodology section present and detailed

* [ ] Sample size and selection criteria documented

* [ ] Limitations acknowledged

* [ ] Heavy data visualization

### Visionary

* [ ] Named senior leader as author

* [ ] Future scenarios grounded in current facts

* [ ] Narrative storytelling present but backed by data

## Final Pre-Publication

* [ ] Spell-check and grammar review completed

* [ ] SME peer review completed

* [ ] All tracked changes resolved

* [ ] PDF renders correctly (if generating PDF)

* [ ] All links work in digital format

* [ ] File size reasonable for distribution

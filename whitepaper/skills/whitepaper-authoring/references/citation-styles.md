# Citation Styles — Detailed Reference

## IEEE (Institute of Electrical and Electronics Engineers)

**Use for:** Engineering, Computer Science, Electrical Engineering, most technical whitepapers.

**In-text format:** Numbered references in square brackets: [1], [2], [3]

**Reference list:** Sorted numerically in order of first appearance.

**Examples:**

In-text: "Cloud-native architectures reduce deployment time by up to 67% [1]."

Reference list entry:
```
[1] J. Smith and A. Johnson, "Cloud-Native Architecture Patterns for Enterprise Systems," IEEE Trans. Cloud Computing, vol. 12, no. 3, pp. 245-260, Mar. 2025.
```

Book:
```
[2] A. Tanenbaum, Computer Networks, 6th ed. London, UK: Pearson, 2024.
```

Website:
```
[3] Kubernetes Authors, "Production-Grade Container Orchestration," Kubernetes.io. [Online]. Available: https://kubernetes.io/. [Accessed: Feb. 15, 2026].
```

White paper:
```
[4] Google Cloud, "Best Practices for Enterprise Organizations on Google Cloud," Google, Mountain View, CA, White Paper, 2025.
```

## APA (American Psychological Association), 7th Edition

**Use for:** Business, social sciences, behavioral technology, general business technology.

**In-text format:** Author-date: (Smith, 2023) or Smith (2023)

**Reference list:** Sorted alphabetically by author last name.

**Examples:**

In-text: "Organizations adopting zero-trust frameworks reported 45% fewer breaches (Johnson & Lee, 2025)."

Journal article:
```
Johnson, R., & Lee, S. (2025). Zero-trust security frameworks in enterprise environments. Journal of Cybersecurity, 18(2), 112-134. https://doi.org/10.xxxx
```

Book:
```
Tanenbaum, A. S. (2024). Computer networks (6th ed.). Pearson.
```

White paper (note the [White paper] designation):
```
Google Cloud. (2025). Best practices for enterprise organizations on Google Cloud [White paper]. https://cloud.google.com/whitepapers
```

Website:
```
Kubernetes Authors. (2026, January 10). Production-grade container orchestration. Kubernetes. https://kubernetes.io/
```

## MLA (Modern Language Association), 9th Edition

**Use for:** Humanities, liberal arts. Rarely used for technical whitepapers but included for completeness.

**In-text format:** Author-page: (Smith 45)

**Works Cited list:** Sorted alphabetically by author last name.

**Examples:**

In-text: "The shift toward distributed systems has fundamentally altered deployment strategies (Smith 45)."

Book:
```
Tanenbaum, Andrew S. Computer Networks. 6th ed., Pearson, 2024.
```

Journal:
```
Johnson, Robert, and Sarah Lee. "Zero-Trust Security Frameworks." Journal of Cybersecurity, vol. 18, no. 2, 2025, pp. 112-134.
```

## Chicago / Turabian

**Use for:** Law, government, policy documents.

**Two systems:**
- Notes-Bibliography (footnotes/endnotes): used in humanities
- Author-Date: used in sciences and social sciences

**Footnote example:**
```
1. Robert Johnson and Sarah Lee, "Zero-Trust Security Frameworks in Enterprise Environments," Journal of Cybersecurity 18, no. 2 (2025): 112-134.
```

**Author-date in-text:** (Johnson and Lee 2025, 112)

## Choosing a Citation Style

Decision tree:
1. Does the target publication or organization mandate a style? Use that.
2. Is this primarily for engineers/CS professionals? Use IEEE.
3. Is this for business/management audiences? Use APA.
4. Is this for government/policy audiences? Use Chicago.
5. No specific audience? Default to IEEE for technical, APA for business.

## Common Rules Across All Styles

- Cite ALL sources used in developing the paper
- Cross-check every in-text citation against the reference list
- Verify all URLs are accessible at time of publication
- Include DOIs where available (they are more stable than URLs)
- Do not cite secondary sources — find and cite the original
- When citing a whitepaper, include the issuing organization, title, and year at minimum

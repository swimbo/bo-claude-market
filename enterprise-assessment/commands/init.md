# Initialize Assessment Configuration

Walk through creating a `.enterprise-assessment.yml` file to customize the enterprise assessment for a specific project.

## Usage
- `/enterprise-assessment:init` — Create configuration interactively

## Instructions

## Step 1: Detect Project

Scan the project to detect:
- Backend language and frameworks
- Frontend framework (if any)
- Existing CI/CD configuration
- Existing security tooling (SAST, dependency scanning)
- Project structure

Report findings briefly.

## Step 2: Select Profile

Ask the user:

> **Which assessment profile fits this project?**
>
> | Profile | Focus | Best For |
> |---------|-------|----------|
> | `default` | General enterprise readiness | Most projects |
> | `fedramp` | Federal compliance (NIST 800-53) | Government workloads |
> | `hipaa` | Healthcare data protection | PHI/ePHI systems |
> | `financial` | SOX, change control, audit trails | Financial services |
> | `minimal` | Security basics only | Internal tools, startups |
>
> Type a profile name, or press enter for `default`.

Wait for response.

## Step 3: Configure Must-Pass

Show the selected profile's default must-pass categories and ask:

> **Must-pass categories for `[profile]` profile:**
> [list profile defaults]
>
> Would you like to adjust? Type category numbers to add/remove, or press enter to keep defaults.

Wait for response.

## Step 4: Configure Weights

Ask:

> **Should any categories have extra weight?** (higher weight = more impact on overall grade)
>
> Profile defaults:
> - Security Posture: 1.5x
> - Access Control & IAM: 1.5x
> - Data Governance: 1.25x
> - Compliance: 1.25x
> - Code Quality: 0.75x
> - Documentation: 0.75x
> - All others: 1.0x
>
> Type adjustments (e.g., "testing: 2.0, ops_readiness: 1.5") or press enter for defaults.

Wait for response.

## Step 5: Disable Categories

Ask:

> **Should any categories be disabled (skipped during evaluation)?**
>
> Common reasons:
> - No frontend → disable "Documentation & Knowledge" API docs check
> - Internal tool → disable "Compliance & Regulatory"
> - No production deployment yet → disable "Operational Readiness"
>
> Type category numbers to disable, or press enter to keep all enabled.

Wait for response.

## Step 6: Custom Categories

Ask:

> **Do you want to add any custom categories specific to your organization?**
>
> Examples: "GDPR Compliance", "Internal API Standards", "Performance SLAs", "Accessibility (WCAG)"
>
> For each custom category, I'll need:
> - Category name
> - A list of checks to evaluate
>
> Type "yes" to add custom categories, or press enter to skip.

If yes, collect category name and checks interactively.

## Step 7: Generate Configuration

Write `.enterprise-assessment.yml` in the project root:

```yaml
# Enterprise Assessment Configuration
# Generated by /enterprise-assessment:init

profile: [selected profile]

must_pass:
  - [category_key]
  - [category_key]

categories:
  # Weight adjustments (only if different from profile defaults)
  # [key]:
  #   weight: [value]

  # Disabled categories
  # [key]:
  #   enabled: false

  # Custom categories
  # [custom_key]:
  #   name: "Display Name"
  #   weight: 1.0
  #   checks:
  #     - "Check description"
```

Only include sections that differ from profile defaults. Keep the file minimal.

## Step 8: Confirm

Tell the user:

> Configuration saved to `.enterprise-assessment.yml` with profile `[profile]`.
>
> Must-pass: [list]
> Disabled: [list or "none"]
> Custom categories: [list or "none"]
>
> Run `/enterprise-assessment:assess` to evaluate this project.
> Edit `.enterprise-assessment.yml` manually at any time to adjust.
